◆ game_move_player()

void game_move_player ( game * g,
short p,
short dir )

Modifie le déplacement de la raquette du joueur spécifié (si il existe).

La fonction devra mettre à jour la vitesse de déplacement de la raquette du joueur spécifié en fonction de la direction donnée en paramètre. Si dir est égal à -1, la raquette doit se déplacer vers la gauche, si dir est égal à 1, elle doit se déplacer vers la droite, sinon elle doit s'arrêter. La valeur de vitesse de déplacement doit être égale à PADDLE_BASE_SPEED (ou à sa valeur modifiée par les bonus) dans la direction indiquée.

Paramètres
gLe jeu.
pNuméro du joueur (0 ou 1).
dirDirection du déplacement (-1 pour gauche, 1 pour droite, 0 pour arrêt).

◆ game_apply_moves()

void game_apply_moves ( game * game)

Bouge tous les objets du jeu selon leur vitesse : les raquettes, les balles, les bonus et aussi les briques quand le bonus de gravité est actif.

Les objets sont déplacés en additionnant leur vitesse à leur position. Cette fonction est appelée par l'interface à chaque tour de la boucle de jeu après avoir pris en compte les actions de l'utilisateur.

Attention
Pour les mouvements des raquettes, il faut veiller à ce que les raquettes ne sortent pas du plateau de jeu.
Paramètres
gameLe jeu.

◆ game_place_bonus_pong()

void game_place_bonus_pong ( game * g)

Ajoute un nouveau bonus au jeu, à une coordonnée horizontale aléatoire au centre du plateau. Utilisé dans le mode Pong pour générer des bonus aléatoires pendant la partie.

Le bonus se déplacera ensuite verticalement vers le bas ou vers le haut selon les joueurs présents dans le jeu :

  • si il n'y a qu'un seul joueur, le bonus se déplacera vers ce joueur,
  • sinon il se déplacera dans une direction aléatoire.

Cette fonction est appelée par l'interface régulièrement pendant la boucle de jeu après game_apply_moves().

Paramètres
gLe jeu.

◆ game_handle_collisions()

void game_handle_collisions ( game * g)

Traite les collisions des balles avec les murs, les raquettes et les briques et les collisions des bonus avec les raquettes.

Cette fonction est appelée à chaque tour de la boucle de jeu après game_apply_moves() pour mettre à jour l'état du jeu en fonction des différentes collisions qui peuvent se produire entre les objets du jeu. Elle supprime également les balles et les bonus qui sortent du plateau de jeu. Elle génère aussi une nouvelle balle si toutes les balles ont été perdues et que le jeu n'est pas terminé.

Remarques
Cette fonction doit appeler ball_hits_wall, ball_hits_paddle et ball_hits_brick pour gérer les différentes collisions.
Paramètres
gLe jeu.
